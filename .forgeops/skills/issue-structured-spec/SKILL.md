# 执行准则

1. 把需求写成可验证的验收标准。
2. 每个 Issue 必须包含范围、非范围、风险、回滚方案。
3. 将任务拆分为可并发执行的小单元并标注依赖。
4. 输出可直接用于 GitHub Issue/PR 模板的结构化内容。
5. 输入不完整时，补齐关键缺口并显式标注 Assumptions（默认不阻塞）。
6. 将用户在 issue/context 中的品位与偏好信号整理为可执行约束。
