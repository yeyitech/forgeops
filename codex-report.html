Core Thesis

Follow, Not Fork + Control Plane First

ForgeOps 聚焦 turn 间编排与治理；turn 内智能由 Codex 上游持续演进承担。

关键判断

  • “任务结束后给下一步建议”主要是指令层行为，不是后处理拼接。
  • Prompt 组装链路：`base_instructions -> developer -> user(AGENTS+skills) -> turn 内 skill 注入`。
  • `AGENTS.md` 是路径级拼接文档并受字节预算限制，适合地图，不适合百科。
  • Skills 是一等机制，支持显式/隐式触发与权限画像。

职责边界矩阵

ForgeOps Codex
workflow / run-step-session 编排、重试、恢复、证据链 turn 内 prompt 组装、skills 注入、MCP 路由

Runtime Adapter 隔离运行时差异 上游能力演进与协议兼容
Prompt & Skills

上下文注入链路

把“为什么会输出当前行为”变成可解释路径，而不是黑箱猜测。

Step 1 Base Instructions

会话初始化优先级：config -> session_meta -> model default。

Step 2 Developer Layer

权限/沙箱、协作模式、feature 指令按顺序拼装。

Step 3 User Layer

用户指令 + AGENTS 路径拼接 + skills section。

Step 4 Turn Skill Injection

显式提及 `$skill` 触发注入，隐式调用可记录并审批。

Session Mechanics

长会话稳定性：机制收益与风险账本

稳定来自工程机制组合，不是单一算法魔法。

稳定来源（源码证据）

  • 队列驱动状态机：提交通道 `bounded(512)`，串行消费会话操作。
  • 单活跃任务：spawn 新任务前先中断旧任务并清理执行进程。
  • 网络降级：采样失败后重试与退避，WebSocket 可回退 HTTPS。
  • 可恢复持久化：rollout JSONL + resume/fork 重建历史。
  • 资源上限：进程/输出/历史裁剪，防止无限膨胀。

长会话风险（设计约束）

  • 多次 compaction 语义漂移：长期线程准确性会下降，建议必要时切新线程。
  • 压缩有损：local/remote compact 都会过滤或截断历史。
  • token 估算误差：压缩触发时机可能偏早/偏晚。
  • 事件通道无界：慢消费场景存在背压放大风险。
  • 跨模型恢复：恢复时模型不一致会引入性能告警。
Session LiveView Contract

L1-L4 能力分层与接口契约

目标是“看见 Agent 正在如何干活”，而不是只看日志片段。

L1

Metadata

`sessionId/runId/stepId/agentId/threadId/turnId/model/status`

L2

Timeline

标准化事件流，支持历史分页与 SSE tail。

L3

Evidence

命令摘要、文件改动、测试结果、风险信号。

L4

Control

resume/fork/interrupt 与观察接口分权鉴权。

  • GET `/api/sessions/:id`
  • GET `/api/sessions/:id/events?afterId=<id>&limit=<n>`
  • GET `/api/sessions/:id/stream?sinceId=<id>` (SSE)
  • GET `/api/sessions/:id/evidence`
  • GET `/api/sessions/:id/resume-command` (只读命令模板)
  • GET `/api/mobile/sessions/:id/snapshot` (移动端聚合)
Stability Rollout

P0 / P1 / P2 落地路径

先观测，再阈值策略，最后产品化闭环，避免先改核心语义。

P0 · Low Risk

先看得见风险

  • 统一 resume/thread/turn 生命周期事件
  • 固化 session 追踪字段（thread/turn/model）
  • 新增 `runtime.session.risk`（只告警，不阻塞）
  • API/SSE 透传风险事件
P1 · Mid Risk

策略阈值可解释

  • 项目级 `longSessionPolicy` 配置读取
  • `runtime.session.rotate.recommended` 触发
  • resume 失败 fallback 语义标准化
  • 非阻塞“建议新开线程/拆小任务”
P2 · Product Loop

进入运营闭环

  • UI 会话健康度（normal/risk/rotate）
  • 风险自动汇总技术债追踪
  • adapter 声明稳定性能力
  • resume/fallback/risk 烟测脚本

推荐默认阈值（文档约定）：`maxResumeFailuresPerStep = 2`，`maxRetriesPerStep = 3`， `sessionRiskWindowMinutes = 30`。当前项目真实启用状态：待补充（需以 `docs/quality/verification-status.md` 核对）。
